# Sprint 5: Incident Response Plan, Capybara Unlimited

Type: IR Planning Document  
Client: Capybara Unlimited  
Framework: NIST SP 800-61 Incident Response Lifecycle

## Objective

Develop a complete Incident Response Plan for Capybara Unlimited covering preparation, detection and analysis, containment, eradication, recovery, and post-incident activity.

## Plan Structure

### 1. Preparation
IR team roles and responsibilities. Communication plan covering internal and external stakeholders. Tools, runbooks, and playbooks inventory. Training and tabletop exercise schedule.

### 2. Detection and Analysis
Alert sources (SIEM, EDR, user reports, threat intel). Triage criteria and severity scoring. Initial scoping methodology.

### 3. Containment, Eradication, and Recovery
Short-term containment to isolate affected systems. Long-term containment with temporary fixes. Eradication to remove root cause, malware, and persistence. Recovery to restore systems and monitor for re-compromise.

### 4. Post-Incident Activity
Lessons learned meeting requirement. Documentation and metrics. Plan update cadence.
